第13步：数据一致性与并发（需求 5）
═══════════════════════════════════════════════════
演示方式：代码讲解
→ app.py 中 borrow() 函数：
  ✅ 乐观锁：UPDATE copies ... WHERE id=%s AND version=%s
  ✅ 事务：start_transaction() + commit/rollback
  ✅ 行锁：SELECT ... FOR UPDATE on copies
  ✅ rowcount检查：并发借同一本书时防止覆盖

→ schema.sql 中：
  ✅ copies.version 字段
  ✅ 4个触发器（borrow后审计、fine后恢复状态、学生状态变更、预约重排）

═══════════════════════════════════════════════════
项目技术亮点总结
═══════════════════════════════════════════════════
1. 存储过程 (sp_borrow_book, sp_process_return, sp_daily_overdue_check)
2. 函数 (fn_calculate_fine, fn_can_borrow, fn_current_borrows)  
3. 触发器 (4个: borrow审计、fine恢复、学生状态、预约重排)
4. 事务 (start_transaction + commit/rollback)
5. 乐观锁 (copies.version + WHERE version=)
6. 行锁 (SELECT ... FOR UPDATE)
7. APScheduler (24h定时逾期检查 + 预约过期处理)
8. Flask Session + 自定义装饰器 (admin_required, superadmin_required)
9. Bootstrap Modal + AJAX (借阅/续借/预约交互)
10. Cropper.js 头像裁剪
